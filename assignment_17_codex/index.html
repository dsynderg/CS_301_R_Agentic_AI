1. What Codex Is

Foundations

Codex is a coding agent that can work on tasks in an isolated environment. You can use it from the Codex app, CLI, or other surfaces connected to your account.

Example prompts

  • "Explain the difference between a coding agent and an autocomplete model, in one paragraph."
  • "Given this repository, summarize what the project does and list the main entry points."

Assignment

Ask Codex to audit a small repo and produce a one-page summary. Verify the summary by checking two files yourself.

2. Prompt Basics

Core Skills

Good prompts are specific about scope, constraints, and output format. Treat prompts like a well-written GitHub issue.

Example prompts

  • "Fix the failing test in `tests/api_user_test.py`. Do not change the test file. Explain the root cause."
  • "Refactor `UserService` to remove duplication. Keep public method signatures unchanged and run unit tests."
  • "Add a README section describing local setup. Keep it under 12 lines and use bullet points."

Assignment

Choose a bug or TODO in your codebase and write a prompt that includes: scope, constraints, expected output, and how to verify.

3. Iterative Workflow

Review & Iterate

Treat Codex like a teammate: ask for a plan, review diffs, and iterate. Request tests and ask for alternatives when uncertain.

Example prompts

  • "Propose a plan before coding. After the plan, wait for my approval."
  • "Show me a minimal diff first. If it looks right, then apply it and run tests."
  • "Provide two approaches and explain tradeoffs in 5 bullets."

Assignment

Run a task in two iterations: first ask for a plan, then approve and request tests. Compare the final result to the original plan.

4. Mini Project

Capstone

Combine everything: clear scope, constraints, verification, and iterative review.

Example prompt sequence

  1. "Scan the repo and list 3 improvements that are low risk and user-visible."
  2. "Pick one improvement and write a short plan."
  3. "Implement the plan with a minimal diff and run tests."

Assignment

Ship one improvement end-to-end. Require: a plan, a diff, tests, and a one-paragraph summary of changes.

Prompt Patterns

Use these templates when you are not sure where to start.

Context → Task → Constraints → Output

Context: You are working in a Node app.
Task: Add a health check endpoint.
Constraints: No new deps, keep tests passing.
Output: Diff + test command output.

Plan First

Before coding, outline a 3-step plan.
Wait for approval before applying changes.

Minimal Diff

Show the smallest possible diff.
If approved, apply and run tests.

FAQ

Where can I use Codex?

You can use Codex through the app, the CLI, and other connected surfaces. The exact surfaces and availability can change, so check the latest OpenAI docs.

How should I review its work?

Treat output like a teammate's PR: check diffs, run tests, and ask for revisions when something looks off.

How do I give it project context?

Keep a short, clear set of project rules (like commands to run) and point Codex at your repo. Short context beats long, vague instructions.
